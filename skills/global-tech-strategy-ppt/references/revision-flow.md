# Revision Flow

Use after a first PPT/contact sheet exists. Classify feedback before editing.

## Feedback Routing

- **洞察范围不对:** return to Insight Agenda; ask which themes/sources/customers/competitors to add or remove; research again.
- **证据不足:** return to Insight Research; search current sources, save screenshots/materials, update source notes.
- **观点不准:** rewrite the claim spine; update affected slide titles, red judgments, and insight summary.
- **洞察总结不对:** revise the synthesis model and strategy implication before touching architecture pages.
- **架构方向不对:** return to Strategy Agenda; ask layer names, modules, constraints, and target differentiators.
- **架构竞争力不强:** rewrite right-side points as `<量化目标/范围> -> <技术抓手> -> <能力效果>`.
- **总体构建策略不清:** return to Strategy Agenda; clarify strategic objective, construction principles, and capability pillars.
- **用户场景讲不清:** ask target users, workflow trigger, involved architecture modules, and measurable scenario value.
- **缺关键技术:** add or revise mechanism pages; do not overload the target architecture page.
- **路线图不实:** revise phases, milestones, KPI gates, dependencies, and delivery constraints.
- **策略总结不聚焦:** decide whether the final page should be roadmap, KPI gates, investment priorities, or executive ask.
- **风格不像参考:** re-open reference samples; extract layout and writing form, not domain-specific nouns.
- **文字不准或要交付编辑:** switch to editable or hybrid route for affected slides.

## Revision Response Pattern

Before editing, state:

```text
我会把这次反馈归到 <category>。
保留：<slides/claims>
重做：<slides/claims>
新增资料/搜索：<sources or topics>
输出：<updated pages or deck>
```

Then implement. After editing, render a contact sheet and inspect the changed pages.
