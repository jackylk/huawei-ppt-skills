# Huawei-Style Enterprise PPT Guide

## Visual System

- Background: white or very light gray only.
- Title: black, top-left, no colored title text.
- Subtitle: dark gray.
- Content boxes: Huawei gray outlines, usually `#BFBFBF` or `#D9D9D9`.
- Emphasis: Huawei red, usually `#C7000B`.
- Bottom viewpoint: restrained Huawei-red callout, usually pale-red fill with thin red border, white callout with red left bar, or thin red rule plus black judgment text.
- Avoid: black panels, dark brush strokes, gradients, cyberpunk glow, decorative blobs, page numbers near titles, logos unless user provides them.
- Avoid: thick red borders, large pure-red frames, full red panels, repeated red rectangles, or red containers around normal content.

## Slide Anatomy

1. Black title.
2. Dark gray subtitle.
3. Central diagram, table, matrix, control flow, or architecture stack.
4. Bottom viewpoint/conclusion with restrained Huawei-red treatment.

## Huawei Red Discipline

- Treat red as a precision signal, not a dominant background color.
- Keep red visual area roughly under 5-8% of a dense slide.
- Use gray outlines for normal structure; use red for arrows, selected labels, keywords, and conclusions.
- Prefer pale-red callouts, red left bars, and thin red rules over heavy red boxes.
- Use heavy red only for exceptional warnings.

## Common Layouts

- Three-layer or four-layer architecture stack.
- Left-to-right story flow.
- 2x3 pain-point grid.
- Capability matrix.
- Industry signal landscape.
- Roadmap timeline.
- Input-control-output flow.
- Resource pool scheduler diagram.

## Text Density

For imagegen slides:

- Keep titles under 24 Chinese characters when possible.
- Keep each card to 1 heading plus 1-2 short lines.
- Avoid dense tables with long text.
- Use English technical labels only where they improve recognition: `Agent Pool`, `Token Pool`, `DBay Agent State Cloud`.

For final editable PPT:

- Use generated images as visual references or backgrounds.
- Rebuild critical text as editable PowerPoint text boxes.
